TraceLoupe UI refresh

Six coordinated moves: a device hero that shows what you're looking at, a sidebar reorganized around scans vs. content, bigger icons, layered tinted surfaces instead of pure black & white, the macOS system accent woven through every interactive state, and empty states that always carry an icon.

Try a system accent →

Every mockup on this page re-tints live. In the real app this is read from macOS (NSColor.controlAccentColor) the way NoteSage does it — no picker needed, it just follows System Settings.

What changes

  • Device hero. The top of the sidebar stops being an app-name row. A large centered illustration — the phone under TraceLoupe's loupe — with generous air above and below, device name, model + iOS version, and an Encrypted chip. With no backup open it becomes a dashed "ghost" phone with a Choose-a-backup action.
  • Scans get their own group. Security and Safety move out of the header into a labeled Scans group. They still read as whole-backup operations, but no longer crowd the device identity. Security carries a green status dot when the last scan was clean.
  • Bigger sidebar icons. 16 px → 20 px, with slightly taller rows. The icon set stays lucide — no re-drawing needed, just a size bump in SidebarMenuButton.
  • Layered surfaces. Light mode drops pure white for a soft cool-tinted off-white with a distinctly deeper sidebar; dark mode lifts off near-black and makes the sidebar the darkest layer so content sits forward. Cards float above both.
  • System accent. Active nav item, selection, sent bubbles, primary buttons, focus rings — all keyed to the macOS accent color, with per-theme mixing so it stays legible on both grounds.
  • Empty states with icons. Every EmptyView gets an accent-tinted icon tile — including the plain list-empty ones (Calls, Safari, Apps…) that today render text only.

Implementation notes

  • System accent, the NoteSage way. A Tauri command reads NSColor.controlAccentColor, converts to oklch in Rust, and the frontend sets --accent-system-value on <html>, re-fetching on window focus. Non-macOS and error paths fall back to a baked-in default. The whole mechanism is ~150 lines and can be ported nearly verbatim from note-sage/src-tauri/src/commands/theme.rs + useAccent.ts.
  • Where accent must NOT go. Scan verdicts stay semantic (green clean / red threat), destructive stays red, and body text stays neutral — the accent is for interaction, never for meaning.
  • Hero SVG is one component. <DeviceHero state="open|empty|rail" /> — plain inline SVG driven by the same CSS variables, so it re-tints with theme and accent for free. The loupe is constant across states; only the phone changes.
  • Scans group styling. Moving Security/Safety from SidebarHeader into a SidebarGroup with a label is a small app-shell.tsx refactor; reimport actions and badges carry over unchanged.
  • Icon size. One cva edit in sidebar.tsx plus row-padding; the density system (comfortable/cozy/compact) can also scale sidebar rows if you want it to.
  • Later, if it earns its keep: per-view counts in the sidebar (Messages 12,482), a subtle scan-status ring around the hero loupe, and accent-tinted selection in list rows.
